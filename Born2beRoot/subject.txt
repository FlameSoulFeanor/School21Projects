Этот проект призван познакомить вас с чудесным миром виртуализации. Вы создадите свою первую машину
в VirtualBox(или UTM, если вы не можете использовать VirtualBox) в соответствии с конкретными инструкциями.
Затем, в конце этого проекта, вы сможете настроить свою собственную операционную систему, соблюдая строгие правила.

Этот проект состоит из того, что вы настраиваете свой первый сервер, следуя определенным правилам.
Поскольку речь идет о настройке сервера, вы установите минимум услуг. По этой причине графический интерфейс здесь бесполезен.
Поэтому запрещено устанавливать X.org или любой другой эквивалентный графический сервер. В противном случае ваша оценка будет 0.
Вы должный выбрать в качестве операционной систему либо последнюю стабильную версию Debian(без тестирования/нестабильную), либо
последнюю стабильную версию CentOS. Debian настоятельно рекомендуется если вы новичок в системном администрировании.

Натсроить CentOS довольно сложно. Следовательно, вам не нужно настраивать KDump. Однако SELinux должен быть запущен при запуске,
а его конфигурация должна быть адаптирована к потребностям проекта. AppArmor для Debian также должен запущен при запуске.

Вы должны создать как минимум 2 зашифрованных раздела с помощью LVM. 

Во время защиты вам зададут несколько вопросов о выбранной вами операционной системе. Например, вы должны знать различия
между aptitude и apt, или чем SELinux или AppArmor является. Корчое поймите, чем пользуететсь)

Служба SSH будет работать только на порту 4242. По соображениям безопасности нельзя подключатся с использованием SSH как root.
Использование SSH будет проверено во время защиты путем настройки новой учетной записи. Поэтому вы должны понять, как это работает.

Вы должны настроить свою операционную систему с помощью брандмауэра UFW и, таким образом, оставить открытым только порт 4242.
Ваш брандмауэр должен быть активен при запуске виртуальной машины. Для CentOS вы должны использовать UFW вместо стандартного брандмауэра.
Для его установки вам понадобится DNF.

* Имя хоста виртуальной машины должно соответствовать вашему логину, оканчивающемуся на 42. Вам нужно будет изменить это имя хоста
во время оценки.
* Вы должны внедрить надежную политику паролей.
* Вы должны установить и настроить sudo в соответствии со строгими правилами.
* В дополнение к пользователю root должен присутствовать пользователь с вашим логином в качестве имени пользователя.
* Этот пользователь должен принадлежать к группам user42 и sudo.

Во время проверки, вы должный создать нового пользователя и добавить его в группу

Чтобы настроить надежные пароли, вы должны соответствовать следующим требованиям:

* Срок действия пароля должен истекать через 30 дней.
* Минимальное кол-во дней до изменения пароля будет установлено равным 2
* Пользователь должен получить предупреждающее сообщение за 7 дней до истечения срока действия пароля.
* Ваш пароль должен состоять не менее чем из 10 символов. Он должен содержать заглавную букву и цифру. Кроме того,
он не должен содержать более 3 одинаковых символов подряд.
* Пароль не должен включать имя пользователя.
* Пароль должен состоять не менее чем из 7 символов, которые не являются частью прежнего пароля.
* Конечно, ваш пароль root должен соответствовать этим правилам.

После настройки файлов конфигурации вам нужно будет изменить все пароли учетных записей, имеющихся на виртуальной машине,
включая корневую учетную запись.

Чтобы настроить надежную конфигурацию для вашей группы sudo, вы должны соблюдать следующие треббования:
* Аутентификация с использованием sudo должна быть ограничена 3 попытками в случае неверного пароля.
* Пользовательское сообщение по вашему выбору должно отображаться, если при использовании sudo возникает ошибка
из-за неправильного пароля.
* Каждое действие, использующее sudo, должно быть заархивировано, как входные, так и выходные. Файл журнала
необходимо сохранить в папке /var/log/sudo/ .
* Режим телетайпа должен быть включен по соображениям безопасности.
* Также по соображениям безопасности пути, которые может использовать sudo, должны быть ограничены. Пример:
	/ usr / local / sbin: / usr / local / bin: / usr / sbin: / usr / bin: / sbin: / bin: / оснастка / bin
